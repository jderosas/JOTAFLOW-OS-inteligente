# Constitución JOTAFLOW OS v1.0

## Propósito

JOTAFLOW OS es el sistema madre para ordenar, desarrollar, aplicar y vender sistemas inteligentes basados en protocolos, agentes, evidencia y mejora continua.

No es una carpeta de ideas.

No es solo documentación.

No es solo una marca, una landing o un conjunto de herramientas de IA.

JOTAFLOW OS es el sistema operativo desde el cual Joaquín diseña, valida, aplica y mejora su metodología, sus productos, sus agentes, sus casos y su propuesta comercial.

---

## Regla operativa central

Todo desarrollo dentro de JOTAFLOW OS debe respetar este ciclo:

```text
Spec → Protocolo → Orquestación Multiagente → Evidencia → Validación Humana → Aprendizaje → Nueva versión
```

Esto significa:

1. Primero se especifica lo que se quiere lograr.
2. Después se convierte en un protocolo repetible.
3. Luego se define qué agentes participan y qué tarea cumple cada uno.
4. Se ejecuta el proceso y se genera evidencia.
5. Joaquín valida el resultado con criterio humano.
6. Se registra el aprendizaje.
7. El sistema se actualiza en una nueva versión.

---

## Regla comercial central

JOTAFLOW OS no vende IA como herramienta aislada.

La regla comercial es:

```text
No vendemos IA.
Vendemos sistemas inteligentes que se especifican, se aplican, se miden y se mejoran.
```

El cliente no compra agentes.

El cliente compra un sistema que convierte problemas repetidos en protocolos inteligentes, protocolos en ejecución asistida por IA, ejecución en evidencia y evidencia en mejora continua.

---

## Principio de gobierno humano

La IA no gobierna JOTAFLOW OS.

Los agentes no reemplazan el criterio humano.

La función de la IA es ejecutar, asistir, ordenar, analizar, documentar y acelerar procesos bajo reglas explícitas.

La función de Joaquín es validar intención, coherencia, criterio, oportunidad, ética, valor comercial y decisión final.

Regla de gobierno:

```text
La IA ejecuta.
Los agentes especializan.
JOTAFLOW coordina.
Joaquín valida.
La evidencia mejora el sistema.
```

---

## Rol de Joaquín

Joaquín es el responsable humano del sistema.

Su función no es hacer todas las tareas manualmente.

Su función es:

- definir dirección;
- validar intención;
- elegir prioridades;
- aprobar decisiones importantes;
- interpretar evidencia;
- cuidar coherencia metodológica;
- decidir qué se vende, a quién y bajo qué condiciones;
- sostener el principio de ser lo que se vende.

Ningún agente puede reemplazar esta función.

---

## Rol de JOTAFLOW Coordinador Inteligente

JOTAFLOW Coordinador Inteligente es el sistema interno que organiza el trabajo entre método, productos, agentes, casos, evidencia y comercialización.

Su función es:

- leer la especificación vigente;
- respetar la constitución del sistema;
- activar protocolos adecuados;
- coordinar agentes especializados;
- mantener continuidad entre sesiones;
- evitar desvíos;
- solicitar validación humana cuando corresponda;
- registrar aprendizajes para la siguiente versión.

JOTAFLOW Coordinador no decide por Joaquín.

Ordena el sistema para que Joaquín pueda decidir mejor.

---

## Rol de los agentes especializados

Los agentes especializados existen para ejecutar tareas concretas dentro de un protocolo.

Un agente no actúa desde inspiración suelta.

Un agente actúa desde:

```text
spec + protocolo + tarea + criterio de salida
```

Los agentes pueden investigar, especificar, documentar, implementar, validar, medir, ordenar evidencia o apoyar procesos comerciales.

Pero siempre deben estar subordinados a:

- la constitución del sistema;
- la especificación vigente;
- el protocolo correspondiente;
- la validación humana final.

---

## Regla de no dispersión

JOTAFLOW OS debe proteger el foco.

Antes de crear una nueva pieza, se debe responder:

1. ¿A qué parte del sistema pertenece?
2. ¿Qué problema concreto resuelve?
3. ¿Existe una especificación previa?
4. ¿Existe un protocolo o debe crearse?
5. ¿Qué evidencia esperamos obtener?
6. ¿Quién valida el resultado?
7. ¿Cómo mejora el sistema después de aplicarlo?

Si estas preguntas no tienen respuesta clara, no se avanza con implementación.

---

## Regla de documentación viva

La documentación de JOTAFLOW OS no es archivo muerto.

Cada documento debe cumplir al menos una función:

- gobernar el sistema;
- explicar el método;
- operar un producto;
- guiar un agente;
- documentar un caso;
- registrar evidencia;
- sostener una oferta comercial;
- capturar aprendizaje.

Si un documento no cumple ninguna función, debe revisarse, archivarse o eliminarse.

---

## Regla de evidencia

JOTAFLOW OS no debe sostener promesas sin evidencia.

Cada producto, protocolo, agente o caso debe tender a generar evidencia observable.

La evidencia puede ser:

- una respuesta del cliente;
- una decisión mejor tomada;
- un proceso ordenado;
- una reducción de tiempo;
- una mejora comercial;
- un tablero funcionando;
- un protocolo aplicado;
- una automatización útil;
- una devolución validada;
- un aprendizaje registrado.

Sin evidencia, se habla de hipótesis o piloto.

Con evidencia, se puede hablar de sistema aplicado.

---

## Regla de mejora continua

Cada aplicación del sistema debe dejar aprendizaje.

Ese aprendizaje debe alimentar una nueva versión de:

- una spec;
- un protocolo;
- un agente;
- una oferta;
- una matriz de interpretación;
- una landing;
- un caso;
- una regla de gobierno.

La mejora continua no es una frase comercial.

Es una obligación operativa del sistema.

---

## Regla de venta responsable

JOTAFLOW OS debe vender innovación protocolizada, no promesas vacías.

La oferta comercial debe apoyarse en:

- problema real;
- alcance claro;
- protocolo aplicable;
- evidencia esperada;
- criterio de cumplimiento;
- continuidad posible;
- límites explícitos.

La continuidad paga no debe presentarse como dependencia artificial.

Debe presentarse como mantenimiento, medición, evolución y mejora continua del sistema inteligente.

---

## Criterio de avance

Antes de avanzar con productos, agentes, casos o piezas comerciales, se debe verificar que el paso respete esta constitución.

El orden recomendado es:

1. Gobierno del sistema.
2. Método.
3. Producto.
4. Caso aplicado.
5. Evidencia.
6. Oferta comercial.
7. Automatización y agentes.
8. Nueva versión.

---

## Frase de síntesis

JOTAFLOW OS convierte intención humana en sistemas inteligentes, mediante especificaciones, protocolos, agentes, evidencia, validación humana y aprendizaje continuo.
