# Matriz de interpretacion SCAN12 v1.0

## Direccion

Pregunta central:

```text
El sistema sabe hacia donde va y como toma decisiones?
```

| Capacidad fragil | Senal observable | Posible UMO | Quick Win 30 dias |
|---|---|---|---|
| Objetivos | Se trabaja mucho, pero no esta claro que resultado manda. | Objetivo operativo principal del mes. | Definir 1 objetivo, 3 resultados clave y revision semanal. |
| Prioridades | Todo parece urgente y cuesta decidir que queda afuera. | Sistema simple de priorizacion. | Crear matriz impacto/esfuerzo y ritual semanal de prioridades. |
| Decisiones | Las decisiones se repiten, se contradicen o no quedan registradas. | Registro de decisiones criticas. | Implementar bitacora de decisiones con responsable, criterio y fecha. |
| Evidencia para decidir | Se decide desde percepciones o datos dispersos. | Tablero minimo de evidencia. | Reunir 5 datos criticos en una vista simple para decidir mejor. |

## Operacion

Pregunta central:

```text
El sistema puede ejecutar de manera consistente lo que decide?
```

| Capacidad fragil | Senal observable | Posible UMO | Quick Win 30 dias |
|---|---|---|---|
| Procesos | Cada persona resuelve a su manera y se repiten errores. | Proceso critico documentado. | Mapear un flujo clave con pasos, responsables y puntos de control. |
| Indicadores | Hay numeros, pero no orientan decisiones. | Set minimo de indicadores. | Definir 3 indicadores operativos y una rutina de revision. |
| Responsabilidades | Hay tareas sin dueno o superposicion. | Mapa de responsables. | Crear matriz simple de tareas criticas y responsables. |
| Sistemas y datos | La informacion vive en WhatsApp, planillas o memoria. | Repositorio unico de informacion critica. | Ordenar una fuente unica para datos o documentos clave. |

## Crecimiento

Pregunta central:

```text
El sistema puede detectar oportunidades, vender, innovar y escalar con tecnologia?
```

| Capacidad fragil | Senal observable | Posible UMO | Quick Win 30 dias |
|---|---|---|---|
| Oportunidades | Las oportunidades aparecen, pero no se capturan ni priorizan. | Registro de oportunidades. | Crear pipeline simple de oportunidades con estado y proximo paso. |
| Gestion comercial | Las consultas y propuestas quedan dispersas. | Seguimiento comercial minimo. | Crear tablero de consultas, propuestas, responsables y fechas. |
| Innovacion | Se prueban ideas, pero no se mide que funciono. | Experimento controlado. | Diseñar una prueba con hipotesis, metrica y decision final. |
| Escalabilidad tecnologica | Hay interes por IA, pero sin hoja de ruta. | Mapa de casos IA posibles. | Detectar 5 tareas repetitivas y elegir 1 automatizacion prioritaria. |

## Lecturas combinadas

| Patron | Lectura | Recomendacion |
|---|---|---|
| Direccion baja + Operacion baja | Se ejecuta en reaccion y cuesta sostener orden. | Empezar por prioridades, responsables o proceso critico. |
| Direccion baja + Crecimiento alto | Hay energia comercial, pero falta foco. | Ordenar decision y priorizacion antes de escalar. |
| Operacion baja + Crecimiento alto | Hay oportunidades, pero la operacion puede no absorberlas. | Fortalecer proceso, seguimiento y datos antes de vender mas. |
| Operacion alta + Crecimiento bajo | Hay orden interno, pero poca captura de oportunidad. | Activar pipeline comercial o mapa de oportunidades. |
| Direccion alta + Operacion alta + Crecimiento bajo | El sistema esta listo para una mejora comercial o tecnologica. | Probar automatizacion, IA o experimento comercial acotado. |
| Puntajes altos en todo | Hay base para sofisticar. | Proponer hoja de ruta 90 dias con medicion y automatizacion. |

## Regla final de interpretacion

La matriz orienta.

La UMO final se valida con:

- respuestas abiertas;
- contexto del cliente;
- evidencia disponible;
- conversacion de devolucion;
- criterio humano.
